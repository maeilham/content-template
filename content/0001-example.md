**스케일업(Scale-up)** 은 기존 서버의 CPU·메모리·디스크를 증설하는 방식입니다. 구현이 단순하고 애플리케이션 변경 없이 성능을 올릴 수 있지만, 하드웨어 한계가 존재하고 단일 장애점(SPOF)이 생깁니다.

**스케일아웃(Scale-out)** 은 서버를 여러 대로 수평 확장하는 방식입니다. 이론적으로 무한 확장이 가능하고 가용성이 높지만, 로드밸런서·세션 공유·데이터 일관성 같은 분산 환경 복잡도가 따라옵니다.

## 언제 무엇을 선택할까?

| | 스케일업 | 스케일아웃 |
|---|---|---|
| 적합한 상황 | DB, 레거시 앱처럼 수평 확장이 어려운 경우 | 웹 서버, API 서버처럼 무상태(stateless)한 경우 |
| 비용 | 고사양 장비는 비용이 급격히 증가 | 저사양 장비 여러 대로 비용 분산 가능 |
| 가용성 | 서버 1대 장애 = 서비스 중단 | 일부 서버 장애에도 서비스 유지 가능 |

실무에서는 둘 중 하나만 선택하기보다 **초기에 스케일업으로 빠르게 대응하고, 한계에 가까워지면 스케일아웃 구조로 전환**하는 경우가 많습니다.
