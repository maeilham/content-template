# 매일함 콘텐츠 기여 가이드

## 구조

```
your-repo/
└── content/
    ├── 0001-slug.md
    ├── 0002-slug.md
    └── ...
```

## 파일 이름 규칙

`NNNN-slug.md`

- `NNNN`: 4자리 숫자 (예: `0001`, `0042`). repo 안에서 unique.
- `slug`: 소문자 영문/숫자/하이픈 (예: `goroutine-vs-thread`)
- 숫자 순서대로 발송됩니다.

## frontmatter

```yaml
---
title: "질문 또는 주제"
preview: "메일에 노출되는 1-2줄. 핵심을 요약하거나 호기심을 유발하는 문장."
tags: [go, concurrency]
---
```

| 필드 | 필수 | 설명 |
|---|---|---|
| `title` | ✓ | 메일 제목 및 Discussion 제목으로 사용 |
| `preview` | ✓ | 메일 본문에 노출되는 요약 |
| `tags` | | 분류용 |

## 본문

본문에 답안과 설명을 작성합니다. 마크다운을 자유롭게 사용할 수 있습니다.

> **preview**는 메일에서 노출되는 힌트 역할을 합니다.
> 본문 전체는 메일 본문의 "자세히 보기" 링크를 통해 확인할 수 있습니다.

## 동작 방식

1. 파일이 `main` 브랜치에 머지되면 매일함 서버가 자동으로 감지합니다.
2. 해당 콘텐츠의 GitHub Discussion이 자동 생성됩니다.
3. 구독자에게 메일이 발송됩니다.
4. 제목이 바뀌면 Discussion 제목도 자동으로 업데이트됩니다.
